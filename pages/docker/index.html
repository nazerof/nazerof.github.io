🤔 What is Docker?

Docker is a platform that packages your application and all its dependencies into a "container" - think of it as a lightweight, portable box that runs the same everywhere.

❓ Why Docker Matters

  • No more "works on my machine" - Containers run identically everywhere
  • Instant environment setup - New team members productive in minutes
  • Resource efficient - Unlike VMs, containers share the host OS kernel
💡 Key Insight: Docker containers are NOT virtual machines. They're much lighter and faster!

🛠️ Installation Steps

  1. Download Docker Desktop
    Terminal Commands
    # For Windows/Mac: Visit docker.com/products/docker-desktop # For Linux Ubuntu/Debian: curl -fsSL https://get.docker.com -o get-docker.sh sudo sh get-docker.sh
  2. Verify Installation
    Terminal Commands
    docker --version # Expected: Docker version 24.x.x, build xxxxx
  3. Test with Hello World
    Terminal Commands
    docker run hello-world # This downloads a test image and runs it
  4. Enable Docker Service (Linux)
    sudo systemctl enable docker sudo systemctl start docker sudo usermod -aG docker $USER # Log out and back in for group changes
✅ If you see "Hello from Docker!" - you're ready to go!

🔧 Common Installation Issues

❌ "Cannot connect to Docker daemon"

# Fix: Start Docker service sudo systemctl start docker # Or on Mac/Windows: Open Docker Desktop app

❌ "Permission denied" on Linux

# Fix: Add user to docker group sudo usermod -aG docker $USER newgrp docker # Or logout/login

❌ WSL2 Issues on Windows

Enable WSL2 in Windows Features, install WSL2 kernel update from Microsoft

❌ Virtualization Disabled

Enable virtualization in BIOS/UEFI settings (Intel VT-x or AMD-V)

🎯 Container Fundamentals

Containers are running instances of Docker images. Think of it this way:

  • Image = Recipe/Blueprint (like a class in programming)
  • Container = Actual dish/Instance (like an object)
  • Registry = Cookbook library (Docker Hub)

🔄 Container Lifecycle

States: Created → Running → Paused/Stopped → Removed

📊 Key Concepts

  • Isolation: Each container has its own filesystem, network, process space
  • Layers: Images built in layers, containers add writable layer on top
  • Ephemeral: Containers are temporary by default - data lost on removal

💻 Essential Docker Commands

1️⃣ Pull an Image

# Download nginx web server image docker pull nginx:latest # List downloaded images docker images

2️⃣ Run Your First Container

# Run nginx in foreground (Ctrl+C to stop) docker run nginx # Run in background (detached) docker run -d nginx # Run with port mapping (host:container) docker run -d -p 8080:80 nginx # Visit http://localhost:8080

3️⃣ Manage Containers

# List running containers docker ps # List all containers (including stopped) docker ps -a # Stop a container docker stop [container_id] # Start a stopped container docker start [container_id] # Remove a container docker rm [container_id]

4️⃣ Interact with Containers

# View logs docker logs [container_id] docker logs -f [container_id] # Follow logs in real-time # Execute commands inside container docker exec -it [container_id] bash # Type 'exit' to leave # Copy files to/from container docker cp file.txt [container_id]:/path/ docker cp [container_id]:/path/file.txt .
🎉 Practice: Run MongoDB with mongo-express UI!
# Create network docker network create mongo-network # Run MongoDB docker run -d \ --name mongodb \ --network mongo-network \ -e MONGO_INITDB_ROOT_USERNAME=admin \ -e MONGO_INITDB_ROOT_PASSWORD=pass \ mongo # Run Mongo Express docker run -d \ --name mongo-express \ --network mongo-network \ -p 8081:8081 \ -e ME_CONFIG_MONGODB_ADMINUSERNAME=admin \ -e ME_CONFIG_MONGODB_ADMINPASSWORD=pass \ -e ME_CONFIG_MONGODB_SERVER=mongodb \ mongo-express # Visit http://localhost:8081

🔧 Common Container Issues

❌ Container exits immediately

Check logs to see why:

docker logs [container_id] # Look for error messages

❌ Port already in use

# Find what's using the port sudo lsof -i :8080 # Linux/Mac netstat -ano | findstr :8080 # Windows # Use different port docker run -p 8090:80 nginx

❌ Cannot connect to container

  • Check port mapping: -p hostPort:containerPort
  • Ensure container is running: docker ps
  • Check firewall settings

❌ "No space left on device"

# Clean up unused resources docker system prune -a # Remove stopped containers docker container prune # Remove unused images docker image prune -a

❌ Container name conflicts

# Remove container with same name docker rm [container_name] # Or use auto-generated names docker run nginx # Docker assigns random name

🎨 Understanding Docker Images

Images are the blueprints for containers. They contain:

  • Base OS: Minimal Linux distribution (Alpine, Ubuntu, etc.)
  • Runtime: Language/framework (Node.js, Python, Java)
  • Application Code: Your actual application
  • Configuration: Environment variables, exposed ports

📚 Image Layers

Key Concept: Each instruction in Dockerfile creates a new layer. Layers are cached and reused!

🏷️ Image Naming

Format: [registry/]username/imagename:tag

  • docker.io/library/nginx:latest (full name)
  • nginx:latest (short form)
  • myapp:v1.2.3 (custom versioning)

Hello from Docker! 🐳

'); }); app.listen(3000, () => { console.log('App running on port 3000'); }); EOF

2️⃣ Write Dockerfile

Container Definition
# Use official Node.js image FROM node:18-alpine # Set working directory WORKDIR /app # Copy package files first (layer caching!) COPY package*.json ./ # Install dependencies RUN npm install # Copy application code COPY . . # Expose port EXPOSE 3000 # Define startup command CMD ["npm", "start"]

3️⃣ Build the Image

# Build with tag docker build -t my-node-app:1.0 . # Watch the layers being created! # Each step creates a new layer # List your images docker images # Run your custom image docker run -d -p 3000:3000 my-node-app:1.0 # Visit http://localhost:3000

🔧 Build Troubleshooting

❌ Build fails at RUN command

# Debug with interactive shell docker run -it node:18-alpine sh # Test commands manually first

❌ Image is too large

Solutions: Use alpine images, multi-stage builds, .dockerignore file
# Create .dockerignore node_modules .git *.md .env

❌ Cannot find Dockerfile

# Specify Dockerfile location docker build -f path/to/Dockerfile -t myapp .

🎭 Multi-Container Applications

Docker Compose lets you define and run multi-container applications using a YAML file.

📋 Why Use Compose?

  • Simple: Define entire stack in one file
  • Reproducible: Same environment every time
  • Networked: Containers can communicate by name
  • Scalable: Easy to scale services up/down

🔧 Key Components

  • Services: Container definitions
  • Networks: How containers communicate
  • Volumes: Data persistence
  • Environment: Configuration variables

🛠️ Complete Web Stack Example

1️⃣ Create docker-compose.yml

Docker Compose Configuration
version: '3.8' services: web: build: . ports: - "3000:3000" depends_on: - db - redis environment: - DATABASE_URL=mongodb://db:27017/myapp - REDIS_URL=redis://redis:6379 volumes: - ./src:/app/src networks: - app-network db: image: mongo:latest ports: - "27017:27017" volumes: - mongo_data:/data/db environment: - MONGO_INITDB_ROOT_USERNAME=admin - MONGO_INITDB_ROOT_PASSWORD=secret networks: - app-network redis: image: redis:alpine ports: - "6379:6379" networks: - app-network volumes: mongo_data: networks: app-network: driver: bridge

2️⃣ Essential Compose Commands

# Start all services in background docker-compose up -d # View all running services docker-compose ps # View logs from all services docker-compose logs -f # Scale a service docker-compose up -d --scale web=3 # Stop all services docker-compose stop # Remove everything (containers, networks) docker-compose down

🔧 Compose Troubleshooting

❌ Service can't connect to another service

# Check if services are on same network docker-compose ps docker network ls # Use service name as hostname # Example: mongodb://db:27017 (not localhost:27017)

❌ Port already in use

# Change port mapping in compose file ports: - "8080:80" # host:container

❌ Volume mount permission issues

# Fix ownership sudo chown -R $USER:$USER ./data

💾 Data Persistence

By default, container data is lost when the container is removed. Volumes solve this by providing persistent storage.

🛠️ Using Volumes

# Create and use a named volume docker volume create my-data docker run -d -v my-data:/data nginx # List all volumes docker volume ls # Inspect volume details docker volume inspect my-data # Bind mount (host directory to container) docker run -d -v /host/data:/container/data nginx # Remove unused volumes docker volume prune

🔧 Volume Issues

Common issues include permission problems and volume not found errors.

🌐 Container Networking

Docker provides different network types for container communication.

🛠️ Network Commands

# Create custom network docker network create my-network # Run container in custom network docker run -d --network my-network --name web nginx # List all networks docker network ls # Inspect network details docker network inspect my-network # Connect existing container to network docker network connect my-network [container_id] # Remove unused networks docker network prune

🔧 Network Issues

Check network configuration and container connectivity.

📤 Image Distribution

Container registries store and distribute Docker images.

🛠️ Push to Registry

# Login to Docker Hub docker login # Tag image for registry docker tag my-app:latest username/my-app:v1.0 # Push to Docker Hub docker push username/my-app:v1.0 # Pull from registry docker pull username/my-app:v1.0 # Search Docker Hub docker search nginx

🔧 Registry Issues

Authentication and connectivity problems are common.

Monitor logs, health checks, and resource usage.

🎭 Beyond Docker Compose

Kubernetes (K8s) provides advanced container orchestration with auto-scaling and self-healing.

🛠️ Basic Kubernetes

# Install minikube for local development minikube start # Check cluster status kubectl cluster-info # List all pods kubectl get pods # Create deployment kubectl create deployment nginx-deployment --image=nginx # Expose deployment as service kubectl expose deployment nginx-deployment --port=80 --type=LoadBalancer # Scale deployment kubectl scale deployment nginx-deployment --replicas=3

🔧 K8s Issues

Check pod status and cluster configuration.
